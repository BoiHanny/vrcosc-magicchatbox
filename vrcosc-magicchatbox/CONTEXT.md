# Context: MagicChatbox

This document serves as the glossary and domain map for MagicChatbox. It defines the canonical terms used in the system and their relationships.

## Glossary

### Core Concepts

- **Module**: A discrete integration (e.g., Twitch, Spotify, OpenAI) that manages a specific external data source. A Module is responsible for fetching data, managing its own settings, and generating a displayable string.
- **Output String**: The final, formatted text generated by a Module. This string is typically intended for display within VRChat (via OSC).
- **Settings Provider**: A generic mechanism (`ISettingsProvider<T>`) used by Modules to read and persist their specific configuration.
- **Integration**: The logical connection and data exchange between MagicChatbox and an external service API.
- **Dispatcher**: The mechanism (`IUiDispatcher`) used to marshal updates from background integration threads back to the WPF UI thread.
- **Integration Settings**: Global toggles that determine whether specific Modules are active/running regardless of their individual `IsEnabled` state.
- **Toast Service**: A notification system used to alert the user of integration errors or status changes without interrupting the primary workflow.

## Domain Relationships

- A **Module** uses a **Settings Provider** to configure its **Integration**.
- A **Module** periodically refreshes its data via the **Integration**, updating its internal state.
- When state changes, the **Module** updates its **Output String**.
- The **Module** uses the **Dispatcher** to notify the UI of state changes.
- The **Integration Settings** can override the running state of a **Module**.
